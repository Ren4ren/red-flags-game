## 1. 一個角色 = 一個檔案

- 新檔放在 `js/anthology/<id>.js`，`<id>` 用小寫英文（例：`marcus.js`）。
- 檔案內容是一個全域常數物件 `const EP_<ID> = { ... };`（例：`const EP_MARCUS = {...}`）。
- 物件的 `id` 欄位要等於 `<id>`。

完成後**務必做三步註冊**（見第 5 節），否則角色不會出現在池子裡。

## 2. 角色物件的完整結構

```js
const EP_XXX = {
  id: 'xxx',                       // 小寫英文，與檔名一致
  name: '角色名',                   // 顯示名（角色名保留原文，其餘一律繁體中文）
  cardSub: '年齡 · 一句身份標籤',    // 池子卡片副標
  avatarText: 'X',                 // 頭像顯示的一個字（通常名字首字母）
  avatarStyle: 'background:linear-gradient(135deg,#色1,#色2)', // 頭像底色
  quote: '「招牌台詞。」',           // 池子卡片上的一句話

  profile: {
    sub: '年齡 · 地點或身份',
    photos: [                      // 每個元素是一張照片的「文字描述」（旁白語氣）
      '📷 一句描述。',
      '📷 一句描述。',
    ],
    bio: '自我介紹。\n用 \\n 換行。',
  },

  // 破綻清單：結局回顧時顯示。兩種寫法擇一：
  telltales: [
    { flag: 'xxx', text: '一句話點出這個破綻' },        // 掠食者篇用這種
    { q: '看起來像紅旗的問句？', a: '其實是這樣' },        // 「偽破綻」篇（正常人）用這種
  ],

  beats: [ /* 每一拍，見第 3 節 */ ],

  endings: { /* 每個結局，見第 4 節 */ },

  resolveTraits(endKey, flags, hasTrait) { /* 特質結算，見第 6 節 */ },
};
```

---

## 3. beats（一拍一拍的劇情）

`beats` 是陣列，每個元素是一拍：

```js
{
  status: '畫面狀態列文字（例：配對成功 · 第一晚）',
  messages: [ /* 這一拍自動播放的內容，見下 */ ],
  choices: [ /* 玩家的選項，見下 */ ],
}
```

### messages 的元素（三種，可混用、依序播放）

```js
{ him: '他說的話', time: '晚上10:02' }   // 對方的訊息泡泡（time 可省）
{ note: '旁白／場景。用 \\n 換行。' }      // 灰字旁白（只演不說，不替玩家下判斷）
{ inner: '（玩家的內心話）' }             // 玩家心裡的聲音
```

每則 message 可再加**條件欄位**（控制「讀傷」與「識人之眼」）：

| 欄位 | 意思 |
|---|---|
| `trait: 'guarded'` | 玩家**持有**這個傷才顯示這句（用來「讀傷」） |
| `skipIfTrait: 'guarded'` | 玩家持有這個傷就**隱藏**這句（給沒帶傷的人看的版本） |
| `minInsight: 2` | 識人之眼**達到門檻**才顯示（讓老手看到更多線索）。門檻：1=留心、3=識人、6=老練 |

> 「讀傷」寫法：通用句不加條件（所有人都看到）；針對某個傷的話另外寫一則加 `trait`，疊加上去。參照 `julian.js` 第 1、2 拍。

### choices 的元素（玩家選項）

```js
{
  text: '玩家選項文字',     // 以「（」開頭 = 當成行動敘述（灰字）；否則 = 玩家訊息泡泡
  flag: 'xxx',             // （可選）選了就記下這個破綻，並讓識人之眼 +1
  minInsight: 2,           // （可選）識人達到門檻才會出現這個「觀察選項」
  reply: [ /* message 陣列：選後對方的回應 */ ],
  followup: {              // （可選）回應後再給一輪選項（巢狀對話）
    reply: [ /* messages */ ],
    choices: [ /* 又一組 choices */ ],
  },
  end: 'endingKey',        // （可選）選了就走向這個結局（對應 endings 裡的 key）
}
```

規則：
- 早離開的選項用 `end` 指向「躲過子彈」類結局；越早走，回顧通常越好笑。
- `flag` 同時有兩個作用：記錄破綻（結局清單打勾）＋ 讓識人之眼長一階。

## 4. endings（結局）

`endings` 是物件，key 自取（例：`leave1`、`refused`、`total`）。每個結局可用以下欄位（都可省）：

```js
endingKey: {
  kicker: '結局 · 小標',
  title: '結局大標',
  dodge: { label: '躲過子彈 · 時機', text: '事後揭露。\\n用 \\n 換行。' },
  anatomy: [                       // 「劇本解剖」逐頁列表
    ['第 1 頁 · 名稱', '這一步在做什麼'],
    ['第 2 頁 · 名稱', '...'],
  ],
  absolve: '給玩家的話（卸下羞恥／點出這不是你的錯）。',
  wink: '回眸一句（點破「看見 ≠ 離開」之類）。',
  resource: '求助資源 · 例：113 保護專線（精神暴力）／165 反詐騙',
  showChecklist: true,             // true = 在結局顯示破綻清單（telltales）打勾結果
}
```

> 注意：破綻清單標題目前在 `engine.js` 寫死——`id` 是 `'daniel'` 時顯示「偽破綻」，其餘顯示「破綻清單」。若新角色要用「偽破綻」標題（正常人篇），需請工程在 `finish()` 那行加上該角色 id。

## 6. 特質系統（傷痕／成長／讀傷／逆流）

### 現有特質（`engine.js` 的 `TRAITS`）

| id | 名稱 | cls | 性質 |
|---|---|---|---|
| `seeThrough` | 看穿包裝 | good | 成長 |
| `silent` | 不敢開口 | scar | 傷痕 |
| `guarded` | 防衛心 | edge | 雙面刃 |
| `jumpy` | 草木皆兵 | scar | 傷痕 |
| `trust` | 重新學會信任 | good | 成長（癒合而來） |
| `spoken` | 說得出口 | good | 成長（癒合而來） |
| `selfdoubt` | 我是不是想太多 | scar | 傷痕 |
| `holdfast` | 守得住自己 | good | 成長 |

`cls` 只影響顏色：`good`（正面）/`scar`（傷痕）/`edge`（雙面刃）。

**新增特質**：在 `TRAITS` 加 `id: { name:'顯示名', cls:'good|scar|edge', desc:'一句說明' }`。

### resolveTraits（結局怎麼改寫玩家的卡）

```js
resolveTraits(endKey, flags, hasTrait) {
  // 回傳一個物件，引擎據此改寫玩家的特質卡：
  //   { add:'id', text:'揭曉文字' }                    → 獲得一個特質
  //   { remove:'舊id', add:'新id', text:'...' }        → 癒合（舊傷換成成長，畫面顯示 ✦）
  //   { remove:'舊id', add:'新id', regress:true, text:'...' } → 逆流（退回更重的傷，畫面顯示 ↘）
  //   { add:null, text:'...' }                          → 不變，只給一段話
  if (endKey === 'refused') return { add:'holdfast', text:'...' };
  // 可用 hasTrait('guarded') 判斷玩家帶了什麼傷，給不同結果（這就是「讀傷」的結局版）
  // 可用 flags.size 或 flags.has('xxx') 判斷玩家戳破了幾條
  return { add:null, text:null };
}
```

規則：
- **癒合** = `remove` 舊傷 + `add` 成長特質（不要加 `regress`）→ 顯示 ✦。
- **逆流** = `remove` + `add` + `regress:true`（剛癒合的成長被收割、退回更重的傷）→ 顯示 ↘。
- 想讀玩家帶進來的傷，用 `hasTrait('silent')` 等判斷，回傳不同文字／特質。
